# DiskCleaner Skill

不让 AI 直接碰硬盘的安全清理 Skill。

它把 WizTree 的扫描结果，变成一套更稳的清理与迁移建议：先看真实占用，再做风险分级，再由用户逐项确认是否执行。

这不是一个“让 AI 自己扫盘乱删文件”的工具。  
它真正的定位是：`WizTree 负责扫描，AI 负责理解，用户负责拍板，脚本负责执行。`

## 适合谁

- C 盘变红，但不敢乱删的人
- 普通清理软件只能清表层缓存，想继续往下找空间黑洞的人
- 想把大目录迁到 D 盘 / E 盘，但不想改原路径的人
- 希望整个流程可解释、可确认、可回滚的人

## 核心能力

- 只读 WizTree 导出的 CSV，不让 AI 自己扫硬盘
- 按 `safe / caution / danger / migrate` 四级规则给出建议
- 区分“该删的垃圾”和“更适合迁移的大目录”
- 支持 Junction 迁移，把目录挪到别的盘但保持原路径可用
- 迁移流程包含校验与回滚脚本，不是一次性硬搬

## 整体流程

1. 用户指定要分析的盘符，以及 WizTree 路径
2. 脚本调用 WizTree 扫描并导出 CSV
3. AI 只读取 CSV 做空间分析，不直接遍历硬盘
4. 输出安全清理项、需确认项、禁止项和可迁移项
5. 用户逐项确认后，再执行清理或迁移
6. 可选重新扫描，验证清理结果

## 为什么它更稳

- 真正的磁盘数据来自 WizTree，不来自模型猜测
- AI 不直接接触系统目录，不自己枚举文件树
- 每一步都要用户确认，不默认替用户做决定
- OneDrive 占位文件、系统目录、安装目录这类高风险区域有明确边界

## 为什么它更能清

- 能顺着 WizTree 结果继续追到真正占空间的目录根因
- 能发现普通清理软件不一定主动提示的隐藏垃圾和漂移目录
- 不只会删，还会建议迁移，把“还要继续用”的大目录挪走

## 相关文件

- `SKILL.md`：工作流入口
- `scripts/Invoke-CdiskScan.ps1`：扫描并导出 CSV
- `scripts/Invoke-CdiskClean.ps1`：安全清理
- `scripts/Invoke-CdiskMigrate.ps1`：Junction 迁移

## 一句话总结

比普通清理软件更聪明，比直接让 AI 动硬盘更安全，还能用 Junction 把大目录无痛挪走。
