# DiskCleaner 初始配置要求

这份 skill 真正开始工作前，最少要确认下面这些条件。

## 1. 必要软件

- 已安装 `WizTree`
- 能拿到 WizTree 的可执行文件路径
- Windows 机器可运行 PowerShell 脚本

如果没有 WizTree，这套 skill 不应该直接替代它去扫盘。

## 2. 必要输入

- 要分析的盘符
  - 例如 `C:`、`D:`、`E:`
- WizTree 安装路径
  - 如果路径不确定，需要先由用户确认

## 3. 运行前提

- 允许在本机临时目录输出 CSV
  - 默认在 `$env:TEMP\\CdiskCleaner\\`
- 如果要做迁移，目标盘必须有足够剩余空间
- 如果要迁移的软件目录仍在被占用，迁移前要先关闭相关软件

## 4. 用户确认边界

以下内容不能默认跳过确认：

- 要分析哪个盘
- WizTree 路径是否正确
- 哪些目录要删
- 哪些目录要迁
- 迁移到哪个目标盘
- 是否删除扫描产生的临时 CSV
- 是否重新扫描验证结果

## 5. 安全边界

这套 skill 默认不应该碰这些位置：

- `System32`
- `WinSxS`
- `Program Files`
- `ProgramData`
- `System Volume Information`
- `Windows\\Installer`
- `AppData\\Local\\Microsoft`
- `AppData\\Local\\Packages`

## 6. OneDrive 相关提醒

- OneDrive 目录可能包含云端占位文件
- WizTree 看到的大小不一定等于真实本地占用
- 这类目录需要用户到资源管理器里再确认“占用空间”

## 7. 迁移模式的额外要求

如果要启用 Junction 迁移，还要满足：

- 一次只迁一个目录
- 迁移前完成复制校验
- 迁移后保留回滚脚本
- 默认不要立即删除回滚材料

## 8. 推荐的首次使用顺序

1. 先分析 `C:` 盘
2. 先看 `safe` 和 `caution` 项
3. 再看 `migrate` 项
4. 最后才决定是否执行删除或 Junction 迁移

## 最小启动条件

最小只需要三样：

- 一台 Windows 机器
- 一个明确的目标盘符
- 一个可用的 WizTree 路径

剩下的清理或迁移动作，都应该在用户逐项确认后再继续。
