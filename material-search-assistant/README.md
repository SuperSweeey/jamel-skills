# material-search-assistant

给我文案，我还你一个可直接剪辑的素材包。

`material-search-assistant` 是一个面向视频创作流程的 skill。它不负责帮你写文案，而是接在文案之后，帮你把“配画面”这一步理顺：拆分镜头、整理素材需求、收集素材、补充 AI 配图或 AI 视频提示词，最后输出一套能直接进入剪辑的软件素材包。

## 它解决什么问题

很多内容创作并不是卡在写文案，而是卡在文案写完之后：

- 每一段到底该配什么画面
- 去哪里找更合适
- 该找视频还是图片
- 哪些地方更适合直接补 AI 配图
- 最后怎么整理成剪辑时能直接用的结构

这个 skill 处理的就是这一步。

## 输入和输出

输入是一段中文文案。

输出是一个独立的素材包目录，通常长这样：

```text
material-pack_<主题>/
  视频/
  图片/
  动画/
  ai配图/
  ai视频/
  分镜清单.md
```

这个目录之外不留散落文件。临时文件、中间产物、提示词文件都收进同一个目录里，方便后续剪辑、复查和归档。

## 它怎么工作

整体流程可以理解成 4 步：

1. 读文案，理解每一段在说什么。
2. 把文案拆成更具体的镜头需求。
3. 按镜头整理素材方案：视频、图片、动画、AI 配图、AI 视频 prompt。
4. 把最终结果收成一个可直接剪辑的素材包。

核心不是“搜到相关素材”，而是先把画面需求说清楚，再去找。

## 输出里通常有什么

- 视频素材
- 图片或截图素材
- 动画片段，包括在需要时由 Remotion 生成的动画
- AI 配图 prompt 和对应图片
- AI 视频 prompt
- 一份分镜清单，用来把文案和素材路径对齐

## 适合什么场景

- 文案已经写好了，但不会配画面
- 想做解说视频、口播视频、知识型视频
- 想把 B-roll、截图、补图整理到一个统一流程里
- 想减少手动搜索、手动归类、手动补画面的重复劳动

## 使用方式

把文案给它，然后让它先出分镜和素材清单，再开始实际采集。

推荐的使用顺序是：

1. 先确认整体剪辑风格和画面风格。
2. 让 skill 先输出分镜和素材需求清单。
3. 确认清单没问题，再执行实际采集。
4. 分批检查素材质量，最后统一打包。

## 下载和生成能力

这个版本的视频下载不只限于一条链路。

常见会用到几类能力：

- 视频下载
- 图片或网页截图采集
- 动画生成
- AI 配图
- AI 视频 prompt 撰写

视频下载默认可以走现成的 `universal-transcriber` 流程；如果更适合直接抓取视频，也可以使用 `yt-dlp`。动画部分则可以调用 Remotion，在需要的时候补足时间线、对比图、信息图或流程动画。

## 质量标准

这个 skill 默认会把素材可用性放在前面：

- 视频优先高清版本
- 图片优先清晰、无明显水印、内容准确
- 动画只在确实能增加信息表达时才生成，常见方式是用 Remotion 渲染
- AI 配图和 AI 视频 prompt 要和整条内容的风格保持一致

重点不是“凑齐”，而是“能直接剪”。
