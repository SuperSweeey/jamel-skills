# Infographic Guidelines

Use this file when deciding whether a script segment should be illustrated with an AI-generated infographic instead of sourced screenshots or video.

## When to prefer an infographic

Prefer an infographic when the script segment is mainly:

- explaining a concept
- comparing two systems
- summarizing a workflow
- showing a sequence of steps
- visualizing an abstract relationship
- compressing many facts into one clean frame

Examples:

- "How this AI differs from normal chatbots"
- "How a skill ecosystem works"
- "Why an agent can act on a computer"
- "The tradeoff between power, cost, and risk"

## When not to prefer an infographic

Do not replace real evidence with an infographic when the segment needs:

- proof of virality
- a real public reaction
- a real news headline
- a real platform interface
- a real creator demo

If the script says "people are discussing this everywhere," use real social or news evidence first. An infographic can support the explanation afterward, but it should not replace proof.

## Best use cases in creator workflows

Infographics work especially well for:

- opening explanation after the hook
- mid-video clarification
- transition cards between evidence-heavy sections
- summarizing a complicated mechanism in 1 frame
- converting a dense verbal explanation into something viewers can grasp in 1-2 seconds

## Style direction for this user

The required visual style is:

- Modern Minimalist Vector
- Chibi / Cute

Translate that into prompt constraints:

- clean vector shapes
- flat color blocks
- smooth rounded outlines
- simplified details
- adorable chibi robots and people
- modern UI-like layout structure
- readable composition with generous spacing
- no photorealism
- no 3D render look
- no gritty texture
- no cluttered background

## Color and composition guidance

Default guidance:

- bright but controlled palette
- high contrast between foreground and background
- limited number of accent colors
- clear hierarchy
- one main focal point
- large readable labels if text is allowed

## Good infographic targets

### Concept card

Use for:

- "what this is"
- "how it is different"

Visual pattern:

- one central cute robot or agent
- 2 to 4 surrounding labeled capability bubbles

### Comparison card

Use for:

- old way vs new way
- human workflow vs agent workflow

Visual pattern:

- split-panel layout
- left/right comparison
- matching icons and mirrored composition

### Workflow card

Use for:

- step-by-step mechanism
- how a skill gets shared or reused

Visual pattern:

- 3 to 5 connected steps
- arrows
- cute robot or human guiding the path

### Tradeoff card

Use for:

- benefits vs risks
- power vs reliability

Visual pattern:

- balanced scale
- contrasting icon groups
- one clear takeaway

## Prompt writing rules

Each prompt should include:

1. scene type
2. core message
3. composition
4. character style
5. vector style constraints
6. what to avoid

Each prompt should avoid:

- long paragraphs inside the image
- generic "tech futuristic" clutter
- hyper-detailed UI mockups that will not read well on video
- complicated backgrounds that distract from the main idea

## Budget discipline

When image generation is paid or limited:

1. Generate only when the infographic will genuinely improve clarity.
2. Prefer one image per selected segment.
3. Skip segments already well-covered by strong real screenshots or real video.
4. Prioritize concept, workflow, comparison, and tradeoff cards over decorative filler.
5. Render sequentially and keep a small batch by default.
